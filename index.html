How to Run your Web Application on the AWS cloud

January 2018

This is how I was able to launch my HBC Tech Scheduler on an AWS Ubuntu Instance in order to use Amazon's free server-side resources.

Creating your Instance

First, make you sure you have an AWS account, logged in as the root user with free trial privileges.

  1. Select Amazon Machine Image type. My computer runs on Ubuntu, so I chose the 64-bit Ubuntu Server 16.04 LTS (HVM), SSD Volume Type.
  2. Choose the t2.micro free-tier eligible instance.
  3. Allow default instance details, storage volume, and tags.
  4. Configure security group: Add a custom TCP rule which specifies a port range that includes the port number your app uses on your localhost. Also make sure to have an SSH rule to allow inbound traffic on TCP port 22.
  5. Review and launch.
  6. Select or create new SSH key pair.

Connecting to your Instance

  1. Go into the directory containing the SSH .pem file.
  2. Make sure the permission of the file is correct
  3. Use the SSH command to connect to your instance. Open up the Connect window, where you can copy and paste Amazon's automatically generated SSH command.

Launching the App

Once your inside your instance,

  1. Git clone your repository
  2. Install all your packages.
  3. Run the program. In order to leave the programming running even after disconnecting from the instance, type: nohup python main.py &. Also, make sure to always keep the instance running.
Now, your web app will be running at the public IP address and specified port number!

Simple Docker Tutorial

January 2018

How to transfer a docker container configured to run my HBC web app to a ubuntu Amazon EC2 Instance

Transferring the Docker Container to the Instance

Note, there is no real benefit to using Docker for my HBC web app since the application environment is very simple. However, I thought it might be useful to get familiar with Docker and the general idea of Linux containers for future purposes.

  1. Create the initial docker container using a standard Ubuntu image "ubuntu". I mapped my computer's home directory to the root directory of the instance.
    docker run -it --name initial -v ~:/root ubuntu
  2. Start container.
    docker start -i initial
  3. Install dependencies in the container necessary to run the application and exit the container.
  4. Convert container to image. (Here I've given the image the name "hbc_app")
    docker commit -m "message" CONTAINER_ID hbc_app
  5. Save the docker image. I have saved the file with name "hbc_web.dk" in the same directory. The file extension does not matter but it is typically a .tar file.
    docker save -o hbc_web.dk hbc_app
  6. Secure copy image to the Amazon EWS instance. The name of the SSH key is "key.pem" and the last argument is the target host.
    scp -i ~/ssh/key.pem hbc_web.dk ubuntu@ec2-34-207-173-18.compute-1.amazonaws.com
  7. Load the Docker file.
    docker load -i hbc_web.dk

Creating the Container on the Instance

Now, you have a docker image on the AWS instance, which you can view with the command docker images

Once connected to the Instance,

  1. Add the app code to the root directory.
  2. Create a docker container from the image, mapping the volume to the root directory and mapping the port 5001 on my localhost where the Flask app runs to the port 5001 on the instance host.
    docker run -it --name -v ~:/root -p 5001:5001 bash
  3. Start the container and run the app.

Web Design | Graphic Design

Background: Datamatch is a semi-satirical Valentine's Day matchmaking service used by over half of Harvard's student body. Students take a quiz and then receive a list of their ten most compatible matches based on our algorithm.

Process: How can you improve an already functional and visually appealing product?
I remember as a freshman, when I first took the quiz, I wanted to know who built the website because I thought it was so cool, but I couldn't find the information I was looking for. So after joining the team in 2016, when our project lead asked us for ways to improve the fifteen-year-old service, I knew what to say. I pitched the idea of a credits page to him, and he immediately took a liking. I mocked it up, first on a whiteboard and later on Photoshop using my favorite family of course-- the Kardashians. I then helped implement it on the website, modifying the HTML and SASS files of our homepage.

Design Thinking | Rapid Product Prototyping | UX Design

In this course Design Survivor: Experiential Lessons in Designing for Desirability, we were given a design challenge each week and had a week to come up with a solution.

I received distinctions (top 10%) for:

  • Challenge Half: Design engaging content for a specific target population

    • Prompt: Design a news app for an older, semi-literate population in rural India with access to basic Internet.

    Solution: A simple news app called Five that contains five news stories. It's most important includes a loading buffer to preload the text and images of the content, to ensure an uninterrupted user experience when reading the five articles.

    View project here.

  • Challenge Three: Operationalizing and Anticipating Desirability

    • Prompt: How do you motivate people to save money?

    Solution: Recognizing that couples tend to overspend on their weddings, my project partner and I created Engagemint, an online wedding planner and registry that encourages newly engaged couples to budget cost-effectively. Engagemint recommends couples a wedding budget and provides low-cost party-planning options to help them meet their budget. Furthermore, after the wedding, couples have the option to exchange unwanted gifts for cash. Engagemint effectively solves the problem of overspending by encouraging users to invest in their marriage, literally and figuratively!

    View project here.

  • Challenge Four: Motivating Sustained Learning

    • Prompt: How do you motivate sustained behavior?

    Solution: Elocutio is a web platform designed to improve the user's communication skills. With the slogan “Say ‘Hello’ to Elo!” Elocutio’s main functionality is its advanced artificial intelligence bot, aptly named Elo, which can engage the user in casual and formal conversation as well as provide valuable feedback to improve the user’s performance.

    In order to combat the huge user attrition rates that online learning platforms suffer from, Elocutio can be synced up with one's mobile calendars, so Elo nudges users to practice with it before an important meeting or presentation. Elocutio can also be integrated with voice-activated devices; the user simply needs to say "Hello Elo!" in order to activate Elocutio in his or her home.

    View project here.

  • Challenge Seven: Representing the User Accurately

    • Prompt: How can you accurately design for self-presentation?

    My partner and I looked at an existing model of presentation, Harvard Q Guide for course evaluations. In our research, we found that the students were most interested in seeing grade distributions. and many of them often looked at hundreds of courses when selecting their courses for the semester.

    Given the sheer volume of classes to browse through (and the clicks and page loads that would require), we decided to create the R-Guide, which is essentially Tinder for classes. On the R-Guide, users can swipe through courses quickly and save them in their CourseCart. Another important feature is our user-generated grade distributions, based on anonymously submitted grades from our user base.

    View project here.

  • Challenge Nine: Designing Counter-Cultural Headphones

    • Prompt: How do you create a counter-cultural user experience?

    My project group decided to focus on the trend of DIY, specifically looking back to the era of mixtapes. The mixtape was a wonderful gift to give to your friends, encapsulating the hours you spent in creating that personal playlist, which the giftee could tangibly appreciate through both the physical mixtape and the playlist itself. So we created ReMix, a headphone and app duo that recalls the DIY mixtape culture of past years while refreshing the process for the modern era and the maker movement.
